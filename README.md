# NodusAI MCP Server

> AI-Powered Signals for Prediction Markets — accessible to any AI agent via MCP.

AI agents connect to this server to get Oracle signals for [Polymarket](https://polymarket.com) and [Kalshi](https://kalshi.com) prediction markets. Signals are generated by Gemini 2.5 Flash with real-time web grounding. Payment is handled natively via the x402 protocol (USDC on Base).

---

## How it works

```
AI Agent  ──MCP──►  NodusAI MCP Server  ──►  nodusai.app/api/prediction
                          │                        │
                          │                   Gemini 2.5 Flash
                          │                   + Web grounding
                          │
                          └──►  Query Registry (every signal logged)
```

1. Agent calls `nodus_get_signal` with a Polymarket or Kalshi URL
2. Server validates USDC payment (x402 / EIP-3009 on Base)
3. Oracle queries Gemini 2.5 Flash with live web search grounding
4. Signal returned: probability, confidence, reasoning, sources
5. Query logged to NodusAI registry
6. Session token issued for free follow-up queries (24h)

## Deploy in 5 minutes (no local setup needed)

### Option 1 — Railway (recommended, free tier available)

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/new/template)

1. Fork this repo on GitHub
2. Go to [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo**
3. Select your fork
4. Add environment variable: `NODUSAI_API_BASE` = `https://nodusai.app`
5. Railway auto-detects `railway.json` and deploys
6. Copy your Railway URL (e.g. `https://nodusai-mcp.up.railway.app`)

That's it. Your MCP server is live.

## Connect AI agents to your deployed server

Once deployed, agents connect with just your URL — no install required.

### Claude Desktop

File: `~/Library/Application Support/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "nodusai": {
      "url": "https://YOUR-DEPLOYMENT-URL/sse"
    }
  }
}
```

Restart Claude Desktop. NodusAI tools appear automatically.

```javascript
const client = new Client({ name: "my-agent", version: "1.0.0" }, { capabilities: {} });

// Check pricing

// Get a signal
const signal = await client.callTool({

  arguments: {
    marketUrl:      "https://polymarket.com/event/...",
    xPaymentHeader: "<base64-eip3009-signed-usdc-auth>",
    walletAddress:  "0xYourBaseWallet",
  }
});
```

## Available tools

| Tool | Description | Payment |
|------|-------------|---------|
| `nodus_pricing` | View USDC prices and session packages | Free |
| `nodus_get_payment_requirements` | Get EIP-3009 payment details for a market | Free |
| `nodus_get_signal` | **Query Oracle for a signal** | 0.10 USDC |
| `nodus_create_session` | Buy session package (multi-query, 24h) | 0.50–15 USDC |
| `nodus_query_with_session` | Query using existing session token | Free (session) |
| `nodus_verify_signal` | Audit grounding sources of a past signal | Free |
| `nodus_query_history` | Your recent queries from the registry | Free |
| `nodus_admin_stats` | Platform-wide stats | Admin |
| `nodus_mock_payment` | Generate mock payment header for dev | Dev only |

## Payment model

- **Network:** Base (Coinbase L2)  
- **Token:** USDC  
- **Protocol:** x402 / EIP-3009 `TransferWithAuthorization`  
- **Non-custodial:** NodusAI never holds funds. Payments go directly on-chain.

| Option | Cost | Best for |
|--------|------|----------|
| Per query | 0.10 USDC | Occasional use |
| Starter session | 0.50 USDC / 10 queries | Testing |
| Pro session | 2.00 USDC / 50 queries | Regular agents |
| Power session | 5.00 USDC / 150 queries | Active trading bots |
| Enterprise session | 15.00 USDC / 600 queries | High-frequency agents |

## Auto-deploy on push

The repo includes a GitHub Actions workflow (`.github/workflows/deploy.yml`) that:
1. Runs all tests on every push and PR
2. Auto-deploys to Railway on merge to `main`

To enable auto-deploy:
1. Go to your Railway project → Settings → **Tokens** → generate a token
2. Add it to your GitHub repo: **Settings → Secrets → Actions** → `RAILWAY_TOKEN`

Every `git push` to `main` now tests and deploys automatically.
